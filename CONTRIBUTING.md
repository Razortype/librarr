# Contributing to librarr

Thank you for considering contributing! A few notes before you dive in.

## Before submitting a PR

1. **Open an issue first** for non-trivial changes. Saves time on both sides.
2. **Keep PRs focused.** One concern per PR. Smaller is faster to review.
3. **Match the existing style.** Run `ruff` and `prettier` before pushing.
4. **Tests for new logic.** If you add a feature, add a test.

## On AI-assisted contributions

This project is built with AI-assisted coding tools. Contributors using AI tools are welcome — just review the output yourself before submitting. PRs that look like raw model output (irrelevant comments, hallucinated APIs, fake imports) will be closed without comment.

## Code of conduct

Be kind. Disagree on technical merits, not on people. Maintainers reserve the right to remove comments and contributors who don't follow this.
